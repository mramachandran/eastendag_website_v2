# Getting the site into your repo

Repo: **https://github.com/mramachandran/eastendag_website_v2**

> Heads up: I can't push to your repo for you — that needs your GitHub login, which I don't handle. But both options below take only a couple of minutes. **Option 1 needs no command line.**

The only file GitHub Pages truly needs is **`index.html`**. Everything else is optional/nice-to-have.

---

## Option 1 — Upload in the browser (easiest, no terminal)

1. Open **https://github.com/mramachandran/eastendag_website_v2**
2. Click **Add file → Upload files**.
3. Drag in **`index.html`** (and, if you like, `README.md`).
4. Scroll down, click **Commit changes**.
5. To add the optional server function: click **Add file → Create new file**, type the name **`api/social.js`** (typing the `/` creates the `api` folder), paste in the contents of `api-social.js`, and commit.
6. Turn on the site: **Settings → Pages → Source: Deploy from a branch → `main` / `/ (root)` → Save.**
7. Wait ~1 minute, then visit **https://mramachandran.github.io/eastendag_website_v2/**

Done. Send that link to Pastor Dave.

---

## Option 2 — Terminal (git)

Run these in the folder where these files are saved (Git Bash or PowerShell on Windows). They clone the empty repo, copy the files in, and push:

```bash
git clone https://github.com/mramachandran/eastendag_website_v2.git
cd eastendag_website_v2

# copy the site in (adjust the source path to where the files live)
cp ../index.html .
cp ../README.md .
mkdir -p api && cp ../api-social.js api/social.js

git add .
git commit -m "Add East End redesign demo site"
git push origin main
```

If `git push` says the branch is `master` not `main`, use `git push origin master` (or rename with `git branch -M main` first).

Then enable GitHub Pages: **Settings → Pages → Deploy from `main` / root.**

## After it's live

- **Custom domain later?** GitHub Pages → Settings → Pages → Custom domain, then point `eastendag.org` DNS at it. (Or host on Vercel for the live social feed — see `DEPLOY-social-feed.md`.)
- **The live FB/IG feed** only runs on a serverless host (Vercel), not GitHub Pages. On Pages the "This Week" strip shows sample tiles automatically — nothing breaks.
